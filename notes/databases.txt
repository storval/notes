en stærsti munurinn er ef þetta er SQL eða NoSQL, 




SQL = structured query language, 
gamli skólinn gott, gilt en gamalt....................td...MySQL......postgreSQL

---table structure og mögulega minna svigrúm fyrir eitthvað óvænt eða nýtt

klassískari uppsetning og vel virta gamla góða kerfið
relational
hægt að vera með tables sem eru td með customer id sem relatar við annað sem er vara keypt og myndar tengsl á milli margra tafla
limited scalability
eins og að byggja skýjagljúfur, scalear upp,
Table Structure




NoSQL= not only structured query language
aðeins mismunandi uppsettning og mism formattað,......td....mongoDB...redis

byggt á td JSON structure og þá stjórnar maður gögnunum á meira fljótandi hátt og hægt að breyta

meira flashy, einfaldara, nútímalegur syntax, 
non-relational
maður ber sjálfur ábyrgðina að tengja saman mismunandi JSON keys og value pairs og gæti verið erfitt að leita í gegnum
gott scalability
eins og að byggja lítil og traust hús, scalear lárétt
